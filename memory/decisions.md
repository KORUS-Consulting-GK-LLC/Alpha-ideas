# Решения

## 2026-07-10 12:57:33 +07:00

- Создан командный каркас `Alpha-Ideas` для будущего GitHub-репозитория.
- Выбрана структура: `ideas/`, `research/`, `plans/`, `artefacts/`, `memory/`, `process/`, `templates/`.
- Статусы идей хранятся в карточках, а не через перемещение файлов.
- Для каждой идеи используется единый `slug` во всех каталогах.
- `.agent/` остается локальной оперативной памятью и игнорируется Git.

## 2026-07-10 13:10:46 +07:00

- Добавлен слой локализации окружения: публичный шаблон `LOCALIZATION.example.md` и игнорируемый персональный файл `LOCALIZATION.md`.
- В `AGENTS.md` явно закреплено: перед использованием локальных путей, SSH aliases, remote-профилей и внешних локальных источников агент должен проверять `LOCALIZATION.md`.
- Решение принято для командной работы: у каждого участника будут свои каталоги, SSH и локальные инструменты.

## 2026-07-10 13:35:04 +07:00

- Добавлена публичная инструкция первичного обживания локального клона: `process/workspace-onboarding.md`.
- Закреплено, что `.agent/` не переносится между участниками; каждый агент создает локальные `.agent/context.md`, `.agent/history.md` и `.agent/agent_score.md` заново.
- Публичные правила больше не предполагают единый абсолютный путь проекта для всех участников: локальный корень определяется по текущему клону и фиксируется локально.
- Устойчивые командные правила, обнаруженные при обживании, должны попадать в `memory/` или `process/`, а не оставаться только в `.agent/`.
