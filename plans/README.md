# Plans

`plans/` хранит планы пилотов и проектов, которые уже прошли первичное обогащение и требуют организованного выполнения.

## Правила

- Для каждой идеи использовать папку `plans/<slug>/`.
- Основной файл плана: `plans/<slug>/plan.md`.
- План создается только после того, как понятны цель, контекст, ключевые риски и ближайший проверяемый шаг.
- План должен быть связан с карточкой идеи и, если есть, с research.
- Если идея вынесена в отдельный проект, в `plans/<slug>/handoff.md` фиксируется путь, статус и оставшийся контекст.
